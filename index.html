HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • h1 represents headers, h1 is always going to be larger than the rest
  • div is used to create a division in a page
  • span creates an inline division
  • ul = unordered list
  • ol = ordered list
  • li = list items
  • img = image via web
  • src = location of image
  • br = break or space in between two blocks of content

CCS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • you can customize a color used by selecting its "hex" # number
  • css sheets are used to customize the look and feel of the site
  • Seperations of Concerns - allowing code to have its own responsibility
  • using a * while working on a css sheet, includes this ruling for ALL elements on the page

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch name: creates a new branch and switches to it

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • (VAR) is old code, but still used - use CONST instead
  • CONST allows for code to be "locked down"
  • After CONST, give it a name
  • = is used to make the assignment
  • each line of code used in JS ends with a ;
  • console.log is code viewed in web browser
  • flow of data - read from top to bottom, and linear
  • JS allows for IF statements
  • Multiple IF statements can be used, IF, ELSE IF
  • ARRAYS are CONST that can hold many items
  • FUNCTION - tells computer how to perform a certain task
  • STRING is for names and must have " "
  • FOR loop allows for a single line of code to be executed over and over
  • Commenting Out - allows for data to be inactive but not deleted, use /* before and */after data you would like inactive
